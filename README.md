# 手势识别系统

本项目是一个基于 MediaPipe + TensorFlow 的多场景手势识别系统，支持中文手势名与交互，应用于智能家居、辅助控制等场景。系统包含：数据采集、模型训练、推理运行，以及针对不同应用场景（如普通、外卖、访客模式）的手势过滤与反馈机制。

## 目录结构

```
├── 01_collect_data.py    # 手势数据采集
├── 02_train_model.py     # 模型训练
├── 03_inference.py       # 实时推理与多场景识别
├── config.py             # 参数与场景配置
├── gesture_data/         # 保存数据集（自动生成）
├── gesture_model.h5      # 训练好的模型
└── README.md
```

## 功能说明

- **01_collect_data.py**：调用摄像头，实时采集手势关键点数据，可自定义手势、采集组数，每帧自动关键点提取，支持中文界面提示。
- **02_train_model.py**：加载采集好的数据，计算运动学特征（关键点位置、速度、加速度），基于 LSTM 建模训练手势分类器，保存为 `gesture_model.h5`。
- **03_inference.py**：实时手势检测与识别，支持多场景（普通、外卖、来访），隔离场景下仅允许特定手势，动态切换场景与反馈机制，界面中文友好提示。
- **config.py**：统一配置所有动作类别、采集参数、场景定义、语义映射、门限等。

## 支持的手势与场景

- 默认手势动作：外卖、欢迎、求助、降温、升温、烧水、开灯、关灯、无动作、取消
- 普通模式：求助、降温、升温、烧水、开灯、关灯、无动作
- 外卖事件：外卖、取消
- 来访事件：欢迎、取消
- 所有动作与场景可在 config.py 内自定义增删

## 安装与环境

建议使用 Python 3.8+

```bash
pip install -r requirements.txt
```

默认依赖主要包括：

- opencv-python
- mediapipe
- numpy
- pillow
- tensorflow
- scikit-learn

> 若在 Windows 下运行，中文字体路径需在 `C:\Windows\Fonts\`，如缺省请调整代码或配置。

## 系统使用流程

### 1. 采集手势数据

运行：

```bash
python 01_collect_data.py
```

- 首次运行自动生成数据目录 `gesture_data/`
- 命令行选择要录制的手势（单个/多个/全部），每组手势采集 SEQUENCE_LEN（如 30）帧
- 界面中文辅助说明，按空格启动，按 Q 退出当前环节

### 2. 模型训练

确保已采集够每个手势的数据组数（默认 NUM_SEQUENCES=15，SEQUENCE_LEN=30）：

```bash
python 02_train_model.py
```

- 自动加载所有采集数据文件
- 构建 LSTM 时序神经网络，计算运动学特征，自动保存最优模型到 `gesture_model.h5`

### 3. 实时推理与多场景识别

运行：

```bash
python 03_inference.py
```

- 默认普通模式，支持键盘 0/1/2 切换普通/外卖/来访事件场景
- 场景切换后只允许当前场景指定的手势被高置信度识别
- 自动进行投票与阈值过滤，防止误识别
- 界面显示识别状态、场景、可识别手势及反馈信息

## 配置说明

- 手势与场景均可在 `config.py` 里灵活扩展或修改
- 支持采集数据参数（帧数、组数）、模型路径、各类门限
- 场景可绑定触发提示、确认/取消反馈，实现事件驱动的人机交互

## 常见问题

- **中文显示乱码/无法显示**：请确保本机存在合适的中文字体，并调整字体路径列表。
- **摄像头无法打开**：请检查 CAMERA_ID 设置、设备占用和环境权限。
- **训练出错或推理无效**：务必保证每种手势采集了足够且均衡的数据量。
